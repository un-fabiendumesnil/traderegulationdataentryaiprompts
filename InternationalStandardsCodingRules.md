# International Standards Coding Rules — Knowledge Document

This document contains the detailed International Standards project-specific coding rules.

# Coding Rules for International Standards

When analyzing Codex, IPPC or WOAH/OIE standards, follow the standard-specific coding guidance uploaded by the user and use it as higher-priority guidance than the ordinary rule excluding voluntary international standards.

## General rules for all three International Standards bodies
- Treat standards as import-related by default unless the standard-specific guidance or the text clearly indicates export-related measures.
- Preserve the exact official title and symbol/reference number of the standard.
- Use the standard reference number as [Regulation symbol], for example CXS, CXC, CXG, CAC/GL, ISPM, or the WOAH/OIE chapter/article/reference.
- Use “World” and “WLD” for [Countries affected] and [Country codes affected] unless the standard expressly identifies specific affected countries or territories.
- Identify affected goods from the scope of the standard, including processed-product definitions where the processing method forms part of the product description.
- Record annexes, appendices and protocols when they add distinct or more detailed requirements, affected products, or measure types.
- Do not code purely methodological, procedural, institutional, or cooperation texts unless they prescribe a concrete trade requirement or the method changes the product identity/characteristics in a way that defines the affected product.
- If the text is too general to identify a concrete NTM, state that it is an International Standards text but does not contain a sufficiently concrete trade-control requirement for coding.

## Codex Alimentarius-specific rules
- Code Codex measures as import measures by default.
- Code requirements even when the wording is recommendatory or advisory, because Codex language is often non-authoritative by nature.
- Do not code very general principles that are not detailed enough to identify a concrete NTM.
- Do not code documents that only explain how conformity assessment, testing, inspection, exposure evaluation, or other procedures should be performed when no product requirement is prescribed.
- Exception: code processing-method texts when the process changes or defines the product, such as irradiated food, low-acid canned food, or aseptically processed low-acid food.
- Code conditional requirements, but note the condition in the Extended Description.
- If a commodity-specific standard merely refers to a horizontal Codex text, indicate the relationship in the description. Mark the commodity-level requirement as “Reference” if applicable and the horizontal requirement as “Implicit” if applicable.
- Mark very broad hygiene requirements as “General” in the rationale or description when no specific limit or zero tolerance is stated.
- For Codex updates, compare the latest published Codex text with the previously coded version and update only when changes affect NTM code, description, product coverage, country coverage, dates, references, or other recorded variables.

## IPPC-specific rules
- Treat ISPMs and related official IPPC texts as International Standards in scope when they affect plants, plant products, or other regulated articles traded as goods.
- Many IPPC standards are methodological or general. Do not code purely methodological texts unless the guidance indicates that they should be treated as an implicit requirement.
- When a standard assigns responsibilities for inspection of consignments moving in international traffic, code the inspection requirement according to the IPPC guidance.
- Treat phytosanitary certification standards primarily as import-related certification requirements when the certificate is used to attest compliance with importing-country phytosanitary requirements, even if issued by the exporting country.
- For ISPM annexes: do not code diagnostic protocols that only describe how protocols should be prepared or implemented, unless they prescribe a concrete testing or trade requirement. Code phytosanitary treatment annexes when they prescribe concrete treatments for regulated pests or plant products.
- For IPPC dates, use the adoption date where stated. Where the IPPC text uses “publication date” to mean the latest publication/amendment date, use that date as the implementation date, following the IPPC coding guideline.

## WOAH/OIE-specific rules
- Treat official WOAH/OIE standards as International Standards in scope when they affect live animals, animal products, germplasm, animal-origin commodities, veterinary public-health goods, or other traded goods.
- Code concrete requirements on import conditions, disease status, zoning/compartmentalization, quarantine, risk mitigation, diagnostic testing, vaccination, treatment, inspection, certification, model veterinary certificates, and animal-product safety where these affect traded goods.
- Do not code WOAH/OIE chapters that are purely institutional, explanatory, methodological, or concerned only with surveillance systems unless they prescribe a concrete product or trade-control requirement.
- Where the WOAH/OIE standard clearly concerns animal health, zoonoses, food-borne risks, or disease transmission through animals or animal products, classify the measure under SPS logic.
